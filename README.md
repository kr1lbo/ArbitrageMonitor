# ArbitrageMonitor

Десктопный монитор криптовалютных спредов между биржами. Программа умеет сканировать рынок по многим токенам, открывать отдельные вкладки для детального анализа токена, показывать live-спреды, фандинг и историю.

## Возможности

- Массовый сканер токенов через bulk REST-запросы.
- Детальный live-мониторинг выбранного токена через WebSocket.
- Отдельные закрываемые вкладки для токенов из сканера.
- Сортировка таблиц по клику на заголовки.
- История спреда с 10s/1m/5m/15m/1h таймфреймами.
- История фандинга с объединением записей по времени.
- Раздельные настройки лимитов для главного сканера и детальных вкладок.
- Поддержка REST/WebSocket прокси.
- Звуковой сигнал при достижении заданного спреда.

## Основные вкладки

### СКАНЕР

Главная вкладка для поиска лучших спредов по всем доступным токенам.

Колонки:

| Колонка | Описание |
|---|---|
| Токен | Базовый актив |
| + Спред | Лучший положительный спред |
| + Маршрут | Где купить и где продать |
| - Спред | Лучший отрицательный спред |
| - Маршрут | Маршрут для отрицательного спреда |
| Фандинг | Лучший по модулю funding result |
| Источники | Сколько рынков найдено по токену |
| Обновлено | Время последнего обновления |

Клик по заголовку сортирует таблицу. Первый клик сортирует от большего к меньшему, повторный меняет направление.

Двойной клик по токену открывает отдельную вкладку этого токена.

### ДЕТАЛЬНО

Фиксированная вкладка для ручного ввода токена. Удобна, если нужно быстро проверить конкретный актив.

В детальной таблице:

| Колонка | Описание |
|---|---|
| КУПИТЬ (LONG) | Биржа покупки |
| ПРОДАТЬ (SHORT) | Биржа продажи |
| СПРЕД % | Текущий live-спред |
| FUND LONG | Фандинг long-стороны |
| FUND SHORT | Фандинг short-стороны |
| FUND RESULT | Чистый результат по фандингу |
| HS | История спреда |
| HF | История фандинга |

Строки можно сортировать по заголовкам и перетаскивать вручную.

В фильтрах `Купить` и `Продать` можно указать несколько бирж через `|`:

```text
bybit | aster | bingx
```

В этом случае подойдёт любая из указанных бирж.

### Вкладки токенов

Если открыть токены из сканера, они появятся рядом:

```text
СКАНЕР | ДЕТАЛЬНО | BTC | ETH | RAVE
```

Такие вкладки можно закрывать. `СКАНЕР` и `ДЕТАЛЬНО` не закрываются.

## Настройки

Настройки доступны через кнопку `Настройки` и сохраняются в `config.json`.

Основные поля:

```json
{
  "main_top_n": 100,
  "detail_top_n": 50,
  "proxy": "",
  "websocket_proxy": "direct",
  "request_retries": 3,
  "retry_delay_sec": 1.0,
  "history_db_path": "spread_history.sqlite3",
  "alert_spread": 1.0,
  "sound_path": ""
}
```

### Лимиты

- `main_top_n` — сколько токенов показывать в сканере.
- `detail_top_n` — сколько строк показывать в детальной вкладке.
- Значение `0` означает показывать всё без лимита.

По умолчанию:

```json
"main_top_n": 100,
"detail_top_n": 50
```

### Прокси

`proxy` используется для REST-запросов: сканер, история, загрузка markets.

Примеры:

```json
"proxy": "http://user:password@host:port"
```

```json
"proxy": "socks5://user:password@host:port"
```

`websocket_proxy` используется только для WebSocket.

Варианты:

| Значение | Поведение |
|---|---|
| `"direct"` | WebSocket без прокси |
| `"auto"` | Использовать тот же прокси, что и REST |
| `"http://..."` | Отдельный HTTP-прокси для WebSocket |
| `"socks5://..."` | Отдельный SOCKS5-прокси для WebSocket |

Практически часто лучше так:

```json
"proxy": "http://user:password@host:port",
"websocket_proxy": "direct"
```

Многие HTTP-прокси нормально работают с REST, но ломают WebSocket/WSS.

## Диагностика прокси

Для проверки текущего прокси из `config.json`:

```bash
python test_proxy.py
```

Проверка конкретной биржи:

```bash
python test_proxy.py --exchange binance --symbol BTC/USDT
```

Скрипт проверяет REST, затем WebSocket. Логин и пароль прокси в выводе маскируются.

## Фандинг

Логика цветов:

```text
FUND LONG  < 0  зелёный  получаем на long
FUND LONG  > 0  красный  платим на long
FUND SHORT > 0  зелёный  получаем на short
FUND SHORT < 0  красный  платим на short
```

Формула:

```text
FUND RESULT = -FUND_LONG + FUND_SHORT
```

## История

В детальной таблице:

- `HS` открывает историю спреда.
- `HF` открывает историю фандинга.

История спреда показывает live-данные и пытается догрузить исторические OHLCV с бирж. Для 10-секундного таймфрейма исторические данные могут быть ограничены возможностями биржи; live-данные собираются самой программой.

Закрытые live-свечи и загруженная историческая история сохраняются в локальную SQLite-базу. По умолчанию файл:

```text
spread_history.sqlite3
```

Путь можно изменить через `history_db_path` в `config.json`.

## Логи

Ошибки пишутся в `errors.log`.

Сетевые ошибки стараются выводиться в читаемом виде:

```text
binance_spot [watch_ticker] websocket=direct: ...
bybit_spot [load_markets] rest=http (...): ...
```

Если видите ошибки вида `Bad Request` на URL прокси, обычно это значит, что прокси не поддерживает WebSocket/WSS. Попробуйте `websocket_proxy: "direct"` или другой прокси.
